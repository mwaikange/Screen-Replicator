Architecture OverviewThe Ngumus Eye backend runs on Next.js (App Router) deployed on Vercel. There is NO traditional REST API — all data operations use one of two methods:Method	When Used	Mobile ApproachNext.js Server Actions	All mutations (create, update, delete)	Call as `POST /api/*` wrapper endpoints you buildDirect Supabase Client	All reads (queries)	Use Supabase JS SDK or REST API directlyFor mobile (React Native / Expo), you should:Use the Supabase JS SDK (`@supabase/supabase-js`) directly for all database reads.Create a thin API layer in your Replit backend (or call Supabase directly) for all write operations.Maintain the Supabase session token in SecureStore (Expo) or AsyncStorage.Required Headers for All API Calls```Content-Type: application/jsonAuthorization: Bearer <supabase_access_token>apikey: <SUPABASE_ANON_KEY>```---2. Authentication & Session Management2.1 LoginScreen: Login  Trigger: User taps "Sign In" button  Supabase Method: `supabase.auth.signInWithPassword()`User Input:`email` (text, required, valid email format)`password` (password, required, min 6 chars)Request (Supabase SDK):```javascriptconst { data, error } = await supabase.auth.signInWithPassword({  email: "user@example.com",  password: "password123"})```Direct REST alternative:```POST https://<SUPABASE_PROJECT_URL>/auth/v1/token?grant_type=password``````json{  "email": "user@example.com",  "password": "password123"}```Success Response (200):```json{  "access_token": "eyJhbGciOiJIUzI1NiIs...",  "token_type": "bearer",  "expires_in": 3600,  "expires_at": 1234567890,  "refresh_token": "v1:abcdef...",  "user": {    "id": "uuid-here",    "email": "user@example.com",    "email_confirmed_at": "2024-01-01T00:00:00Z",    "created_at": "2024-01-01T00:00:00Z",    "user_metadata": {      "display_name": "John Doe",      "phone": "+264811234567"    }  }}```After success:Store `access_token` and `refresh_token` securely (SecureStore)Store `user.id` for referenceCheck if user has active subscription (see Section 21)Navigate to `/feed` if subscription active, otherwise `/subscribe`Error Responses:`400`: Invalid credentials```json{ "error": "Invalid login credentials" }````400`: Email not confirmed```json{ "error": "Email not confirmed" }```UI Requirements:Disable Sign In button while loadingShow error message inline below formShow/hide password toggle button"Forgot password?" link navigates to forgot password screen---2.2 Session PersistenceTrigger: App launch / cold startAction: Restore session from SecureStore, then verify it's still valid.```javascriptconst { data: { session } } = await supabase.auth.getSession()```If session exists and not expired: Navigate to `/feed`  If session expired: Use refresh token:```javascriptconst { data, error } = await supabase.auth.refreshSession({  refresh_token: storedRefreshToken})```Success Response (200):```json{  "session": {    "access_token": "new_token_here",    "refresh_token": "new_refresh_token",    "expires_at": 1234567890  }}```If refresh fails: Clear stored tokens, navigate to login screen.---2.3 LogoutScreen: Profile / Settings  Trigger: User taps "Sign Out"Supabase Method:```javascriptconst { error } = await supabase.auth.signOut()```After success:Clear all stored tokens from SecureStoreClear any cached user dataNavigate to login screen---2.4 Session Expiry Mid-UseTrigger: API call returns `401 Unauthorized`Action:Attempt token refresh using stored `refresh_token`If refresh succeeds: retry original request with new tokenIf refresh fails: clear session, navigate to login with message "Session expired. Please sign in again."---3. The Supabase SDK handles token storage — configure it to use `AsyncStorage` or `SecureStore`:```javascriptimport * as SecureStore from "expo-secure-store"import { createClient } from "@supabase/supabase-js"const supabase = createClient(  process.env.EXPO_PUBLIC_SUPABASE_URL,  process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY,  {    auth: {      storage: {        getItem: SecureStore.getItemAsync,        setItem: SecureStore.setItemAsync,        removeItem: SecureStore.deleteItemAsync      },      autoRefreshToken: true,      persistSession: true,      detectSessionInUrl: false    }  })```Required Headers for All Supabase RequestsThe Supabase JS SDK handles these automatically when initialized with your URL and anon key:`Authorization: Bearer <access_token>``apikey: <anon_key>``Content-Type: application/json`---End of documentation.
