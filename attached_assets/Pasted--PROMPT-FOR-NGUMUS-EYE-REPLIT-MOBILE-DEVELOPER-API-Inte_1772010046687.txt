## SUPABASE SETUP

### Supabase Client Initialization

```javascript
import * as SecureStore from "expo-secure-store"
import { createClient } from "@supabase/supabase-js"

const supabase = createClient(
  process.env.EXPO_PUBLIC_SUPABASE_URL,
  process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY,
  {
    auth: {
      storage: {
        getItem: SecureStore.getItemAsync,
        setItem: SecureStore.setItemAsync,
        removeItem: SecureStore.deleteItemAsync
      },
      autoRefreshToken: true,
      persistSession: true,
      detectSessionInUrl: false
    }
  }
)
```

### Required Environment Variables

```
EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIs...
EXPO_PUBLIC_SITE_URL=https://app.ngumus-eye.site
```

> **NEVER** include `SUPABASE_SERVICE_ROLE_KEY` or `SUPABASE_JWT_SECRET` in the mobile app.

### Required Headers for All Direct API Calls

The Supabase JS SDK handles these automatically. For any manual `fetch()` calls to `https://app.ngumus-eye.site/api/*`:
```
Content-Type: application/json
Authorization: Bearer <supabase_access_token>
apikey: <SUPABASE_ANON_KEY>
```

## SECTION 1 — AUTHENTICATION & SESSION MANAGEMENT

### 1.1 Login

**Screen:** Login  
**Trigger:** User taps "Sign In" button

**User Input:**
- `email` (text, required, valid email format)
- `password` (password, required, min 6 chars)

**Supabase SDK Call:**
```javascript
const { data, error } = await supabase.auth.signInWithPassword({
  email: "user@example.com",
  password: "password123"
})
```

**Success Response (200):**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "expires_in": 3600,
  "expires_at": 1234567890,
  "refresh_token": "v1:abcdef...",
  "user": {
    "id": "uuid-here",
    "email": "user@example.com",
    "user_metadata": {
      "display_name": "John Doe",
      "phone": "+264811234567"
    }
  }
}
```

**After success:**
- Store `access_token` and `refresh_token` in SecureStore
- Store `user.id` in state
- Check active subscription (see Section 11.1)
- If subscription active → navigate to `/feed`
- If no subscription → navigate to `/subscribe`

**Error Responses:**
- `400` Invalid credentials: `{ "error": "Invalid login credentials" }`
- `400` Unconfirmed email: `{ "error": "Email not confirmed" }`

**UI Requirements:**
- Disable Sign In button while loading (show spinner)
- Show error message inline below form
- Show/hide password toggle button
- "Forgot password?" link navigates to forgot password screen

---

### 1.2 Session Persistence (App Launch)

**Trigger:** App cold start

```javascript
const { data: { session } } = await supabase.auth.getSession()
```

- If session valid → navigate to `/feed`
- If session expired → attempt refresh:

```javascript
const { data, error } = await supabase.auth.refreshSession({
  refresh_token: storedRefreshToken
})
```

- If refresh succeeds → update stored tokens, navigate to `/feed`
- If refresh fails → clear all stored tokens, navigate to Login screen

---

### 1.3 Logout

**Screen:** Profile / Settings  
**Trigger:** User taps "Sign Out"

```javascript
const { error } = await supabase.auth.signOut()
```

**After success:**
- Clear all tokens from SecureStore
- Clear all cached user data from state
- Navigate to Login screen

---

### 1.4 Session Expiry Mid-Use (401 Handling)

**Trigger:** Any Supabase call returns `401 Unauthorized`

1. Attempt token refresh using stored `refresh_token`
2. If refresh succeeds → retry original request with new token
3. If refresh fails → clear session, navigate to Login with alert: *"Session expired. Please sign in again."*

```javascript
// Step 1: Upload to storage
const filePath = `${userId}/${incidentId}/${Date.now()}-${randomString}.${ext}`

  .from("incident-media")
```

```javascript
const expiresAt = new Date()
```

## SECTION 20 — FILE UPLOAD API

**Endpoint:** `POST https://app.ngumus-eye.site/api/upload`  
**Content-Type:** `multipart/form-data`  
**Auth Header:** `Authorization: Bearer <access_token>`  
**Form field:** `file` (binary)

**Constraints:**
- Images: max 10MB, types: `image/*`
- Videos: max 50MB, types: `video/*`

**Success Response:**
```json
{ "url": "https://public.blob.vercel-storage.com/filename-abc123.jpg" }
```

**Error Responses:**
- `400`: `{ "error": "No file provided" }`
- `400`: `{ "error": "File too large. Maximum size is 10MB" }`
- `400`: `{ "error": "Invalid file type. Only images and videos are allowed" }`
- `500`: `{ "error": "Upload failed" }`
